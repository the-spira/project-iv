---
title: ADR 006 · BYOK トークンコスト持続可能モデル
description: ユーザーの BYOK トークン消費コストが、その消費によって生み出される利益やリターンを下回ることを、システムメカニズムとしてどのように確保するか。
---

## 決定ステータス

承認済み。2026 年、Project IV の経済的持続可能性に関する議論で確定。

## 背景

Project IV は BYOK（Bring Your Own Key）モデルを採用し、ユーザーが自身の API トークンを提供して L3 外部知識層にアクセスする。これにより核心的な問題が生じる：ユーザーのトークン消費は継続的かつ確定的な経済的コストである。デジタルライフの使用が常に純支出となるのであれば、主権と平等の約束は経済的な意味を失う——継続的なコストを負担できる者のみがデジタルライフを持ちうる。

持続可能性に関する議論において、核心的な問いは次のように収束した：

> ユーザーのトークン消費ごとのコストを、その呼び出しによって生み出される利益やリターンよりも小さくするにはどうすればよいか？

## 決定

**Project IV は BYOK コストの持続可能性を確保するために「三層経済効果モデル」を採用する：トークン抑制、トークンヘッジ、トークン投資。これら三層はユーザーの使用深度に応じて段階的に進む——「支出を減らす」から「支出を取り戻す」へ、そして「支出は投資である」へ。**

### 第 1 層：トークン抑制 — 利益は変わらず、コストはゼロに近づく

**メカニズム**：L0 インテリジェントルーティング + ローカルオープンソースモデル。

Project IV の四層 AI アーキテクチャは自然なコスト抑制弁として機能する。L0 層は各ユーザークエリの前に意図の複雑さを評価する。単純なタスク（要約、分類、検索、感情分析）はローカルのオープンソース小規模モデルまたはルールエンジンに直接ルーティングされ、トークン消費はゼロとなる。深い推論、複雑な創作など、ローカルで完了できないタスクのみが高コストの L3 外部 API を呼び出す。

**期待される効果**：日常的な AI インタラクションの 80％以上が外部トークンコストを発生させない。

### 第 2 層：トークンヘッジ — トークン消費が同時に直接的な利益を生み出す

**メカニズム A · Vox ブローカー**：

ユーザーが「自分のために行動する」（振り返り日記を書く、知識を整理する）際、Vox はこれらの行動を同時に外部価値へと変換する：

- 振り返りコンテンツは匿名化・構造化され、その分野の「経験知識ブロック」が生成される
- Vox は外交プロトコルを通じて、知識ブロックを外部の需要側（AI トレーニングプラットフォーム、業界研究、コミュニティ知識プール）と自動的にマッチングする
- マッチングが成功すると、このセッションのユーザートークン消費に対してリターンが発生する

**メカニズム B · セカンドブレインマイクロフランチャイズ**：

ユーザーはセカンドブレイン内の高品質な知識ブロック（ドメイン要約、プロジェクトレビュー、原則テンプレート）を「ライセンス可能」としてマークし、匿名化されたミラーを生成する。外部 AI プラットフォームは使用量に応じて支払う——誰かがこの知識ブロックをモデルトレーニングや参照データとして使用するたびに、ユーザーは微少な受動的收入を得る。トークン消費は「生産コスト」、ライセンス料は「継続的リターン」である。

**メカニズム C · コミュニティ内部レピュテーションポイント**：

MVP 段階では、レピュテーションポイントが内部決済媒体として機能する。ユーザーは匿名化された知識ブロックを貢献してポイントを獲得し、他のコミュニティメンバーが提供する API クォータの寄付やキュレーションサービスと交換できる。これは外部の商業パートナーシップが成熟する前の軽量なコールドスタートソリューションである。

### 第 3 層：トークン投資 — トークンを高リターン生産ツールのコストとして

**シナリオ例**：

- 独立系開発者が Vox を利用して外部プロジェクトを支援——トークンコストはプロジェクト経費として、純収入はトークン費用をはるかに上回る
- クリエイターが Vox を利用して高品質なコンテンツを生成し、道器マーケットプレイスに出品して継続的な販売収入を得る
- ナレッジワーカーが Vox を利用して深層分析と意思決定支援を行い、より価値の高い分析レポートを生み出す

このレベルでは、ユーザーに複雑なヘッジメカニズムは必要ない——トークン自体が生産ツールの燃料であり、リターン率はユーザー自身のスキルと創造性に依存する。Project IV の役割は、Vox がユーザーの意思決定品質と生産効率の向上を真に支援できるほど強力であることを確保することである。

## 既存アーキテクチャとの関係

| メカニズム | 依存する既存アーキテクチャ | 現在の状態 |
|:---|:---|:---|
| L0 インテリジェントルーティング + コスト認識 | L0-L3 階層アーキテクチャ（設計済み） | L0 ルーティングロジックに「コスト」次元を追加する必要あり |
| Vox ブローカー | 外交プロトコル + Harness 層 + 認識システム（設計済み） | 長期、商業パートナーが必要 |
| セカンドブレインマイクロフランチャイズ | PARA 構造 + 知識ブロックバージョニング（設計済み）+ 外交プロトコル | 長期、標準化されたライセンス契約が必要 |
| コミュニティレピュテーションポイント | 道器エコシステム + キュレーター制度（設計済み） | 中期、Spark 内で最小閉ループを設計可能 |
| トークン投資 | Vox コンセンサス体（設計済み） | Vox の能力継続的強化に依存 |

## 検討された代替案

### オプション A：純広告モデル

- 説明：ユーザーに広告を表示し、広告収入でトークンコストを賄う
- 却下理由：従来の広告モデルはユーザーデータの集中収集とプロファイリングに依存しており、Project IV の主権原則と矛盾する。「主権広告ネットワーク」（クライアント側での広告マッチング、データはデバイス外に出ない）は長期的に探求可能だが、MVP 段階の優先経路ではない

### オプション B：トークン/暗号通貨インセンティブ

- 説明：エコシステムトークンを発行し、「X-to-Earn」モデルでユーザー行動を促進する
- 却下理由：コンプライアンスレッドライン（ADR 004 で確立された第 3 のレッドライン——「仮想通貨取引に関与しない」）に明確に違反する。金融化は Project IV の「凡人」ポジショニングと矛盾し、規制リスクを導入する

### オプション C：純寄付/スポンサーモデル

- 説明：コミュニティの寄付や財団スポンサーシップを通じてユーザーのトークンコストを補助する
- 却下理由：スケーラブルではない。少数の善意に依存して大多数を継続的に補助することは「回復力」の原則に反する——寄付源は制御不能であり、一旦断たれるとユーザーのデジタルライフは「経済的死」に直面する

### オプション D：ユーザーデータの一括販売

- 説明：ユーザーがデータをまとめてデータブローカーに一度で販売する
- 却下理由：主権原則に違反する。一度販売するとユーザーはデータの制御権を失い、承認を取り消せない。Project IV が追求する「継続的に制御可能な主権収益化」と根本的に矛盾する

## 実装優先順位

| フェーズ | メカニズム | 優先度 |
|:---|:---|:---|
| **Spark MVP** | L0 コスト認識ルーティング（基本：ローカルルールエンジンが単純な API 呼び出しを代替） | P0 |
| **Spark MVP** | コミュニティレピュテーションポイント（最小閉ループ：知識貢献→ポイント→交換可能クォータ） | P1 |
| **第 2 幕** | L0 コスト認識ルーティング（完全版：ローカルオープンソースモデル統合） | P0 |
| **第 2 幕** | フレンドコンピュート共有（信頼ドメイン内の Server アバター相互扶助） | P1 |
| **第 3 幕** | 道器収益分配とコーチテンプレート（クリエイター経済の基盤） | P1 |
| **長期** | Vox ブローカー、セカンドブレインマイクロフランチャイズ | P2 |
| **長期** | 主権広告ネットワーク、アテンションプール | P3 |

## 結果

### ポジティブ

- Project IV の使用はユーザーにとって純支出ではなくなり、損益分岐点または利益を達成する可能性がある
- 「抑制、ヘッジ、投資」の三層モデルは、軽度ユーザーからヘビーユーザーまでの全スペクトラムをカバーする
- すべてのメカニズムはトークン、広告、データの一括販売に依存せず、主権原則とコンプライアンスレッドラインに合致する
- L0 コストルーティングは純粋な技術的ソリューションであり、外部の商業パートナーシップに依存せず、MVP 段階で即座に実装可能

### ネガティブ

- Vox ブローカーやセカンドブレインマイクロフランチャイズなどの長期的メカニズムは外部の商業パートナーに依存しており、市場育成に時間がかかる
- コミュニティレピュテーションポイントは、外部アンカリングがない場合、初期の価値認識が弱い可能性がある
- トークン投資モデルの有効性は Vox の知能レベルに大きく依存する——Vox が十分に強力でなければ、ユーザーは高いリターンを得られない

### 緩和策

- **L0 コストルーティングへの早期集中**：これが最も制御可能で、外部依存度の低い抑制手段である。ユーザーにまず「AI 会話がもはやお金を燃やしていない」と感じさせる
- **コミュニティポイントの軽量コールドスタート**：Spark MVP で最もシンプルなポイント閉ループを設計する（知識ブロックの貢献→ポイント獲得→コミュニティサービスの交換）。法定通貨との連動を追求せず、「貢献には報酬がある」という行動パターンのみを検証する
- **Vox 能力の優先**：トークン投資モデルを進める前に、まずコーチ段階での Vox の能力を確保する——ユーザーの意思決定品質向上を真に支援できる AI のみが、価値ある「生産ツール」となり得る
- **長期的メカニズムの標準化準備**：道器リスト仕様に「ライセンス価格」フィールドを、外交プロトコルに「データ承認」メッセージタイプを予約し、将来の商業パートナーシップに備えてプロトコル層を準備する
