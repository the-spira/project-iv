title: ADR 006 · BYOK Token 成本可持续模型
description: 如何从系统机制上确保用户 BYOK token 消耗的费用小于其带来的收益或回报。
---

## 决策状态

已采纳。2026 年，在 Project IV 经济可持续性讨论中确定。

## 背景

Project IV 采用 BYOK 模式，用户自行提供 API token 以调用 L3 外部知识层。这带来了一个核心问题：用户的 token 消耗是持续、确定的经济成本。如果数字生命的使用始终是净支出，那么主权和平权的承诺将在经济层面失去意义——只有能负担持续开销的人才能拥有数字生命。

在可持续性讨论中，核心问题被收敛为：

> 如何让用户每次 token 消耗的费用，小于这次调用所带来的收益或回报？

## 决策

**Project IV 采用"三级经济效益模型"确保 BYOK 成本可持续：Token 节流、Token 对冲、Token 投资。三者按用户的使用深度递进，从"少花钱"到"花出去的钱赚回来"再到"花出去的钱是投资"。**

### 第一级：Token 节流 — 让收益不变，成本趋近于零

**机制**：L0 智能路由 + 本地开源模型。

Project IV 的四层 AI 架构是天然的成本节流阀。L0 层在每次用户提问时先判断意图复杂度。简单任务（摘要、分类、检索、情感分析）直接路由到本地开源小模型或规则引擎，token 消耗为零。只有深度推理、复杂创作等本地无法完成的任务，才调用昂贵的 L3 外部 API。

**预期效果**：80% 以上的日常 AI 交互不产生外部 token 费用。

### 第二级：Token 对冲 — 让 token 消耗的同时产生直接收益

**机制 A · Vox 经纪人**：

用户在"为自己做事"（写复盘日记、整理知识）时，Vox 将这些行为同步转化为对外价值：

- 复盘内容被匿名化、结构化，生成该领域的"经验知识块"
- Vox 通过外交协议自动将知识块与外部需求方匹配（AI 训练平台、行业研究、社区知识池）
- 匹配成功后，用户本次 token 消耗获得回报

**机制 B · 第二大脑微特许**：

用户将第二大脑中的高质量知识块（领域总结、项目复盘、原则模板）标记为"可授权"，生成匿名化镜像。外部 AI 平台按使用量付费——每次有人用这个知识块训练模型或作为参考数据，用户获得微小被动收入。Token 消耗是"生产成本"，授权费是"持续回报"。

**机制 C · 社区内部信誉积分**：

MVP 阶段，用信誉积分作为内部结算媒介。用户贡献匿名知识块获得积分，积分可兑换其他社区成员提供的 API 额度捐赠或策展服务。这是在外部商业合作成熟前的轻量级冷启动方案。

### 第三级：Token 投资 — 让 token 作为高回报生产工具的成本

**场景示例**：

- 独立开发者用 Vox 辅助完成外包项目，token 消耗作为项目成本，净收入远超 token 费用
- 创作者用 Vox 生成高质量内容在道器集市上架，获得持续销售收入
- 知识工作者用 Vox 进行深度分析和决策辅助，产出更高价值的分析报告

在这一级，用户不需要复杂的对冲机制——token 本身就是生产工具的燃料，回报率取决于用户自身的技能和创造力。Project IV 的角色是确保 Vox 足够强大，能够真正帮助用户提高决策质量和产出效率。

## 与现有架构的关系

| 机制 | 依赖的现有架构 | 当前状态 |
|:---|:---|:---|
| L0 智能路由 + 成本意识 | L0-L3 分层架构（已设计） | 需在 L0 路由逻辑中增加"成本"维度 |
| Vox 经纪人 | 外交协议 + Harness 层 + 感知系统（已设计） | 远期，需商业合作方 |
| 第二大脑微特许 | PARA 结构 + 知识块版本化（已设计）+ 外交协议 | 远期，需标准化授权协议 |
| 社区信誉积分 | 道器生态 + 策展人制度（已设计） | 中期，可在 Spark 中设计最小闭环 |
| Token 投资 | Vox 共识体（已设计） | 依赖 Vox 能力的持续增强 |

## 考虑的替代方案

### 方案 A：纯广告模式

- 描述：向用户展示广告，用广告收入覆盖 token 成本
- 否决理由：传统的广告模式依赖用户数据的中心化收集和画像，与 Project IV 的主权原则冲突。远期可探索"主权广告网络"（用户端侧匹配广告，数据不出设备），但不作为 MVP 阶段的优先路径

### 方案 B：代币/加密货币激励

- 描述：发行生态代币，通过"X-to-Earn"模式激励用户行为
- 否决理由：明确违反合规红线（ADR 004 中确立的第三条红线——"不涉及虚拟货币交易"）。金融化手段与 Project IV 的"凡人"定位不符，且引入监管风险

### 方案 C：纯捐赠/赞助模式

- 描述：通过社区捐赠或基金会赞助来补贴用户的 token 成本
- 否决理由：不可规模化。依赖少数人的善意为多数人提供持续补贴，违背"韧性"原则——捐赠来源不可控，一旦断供，用户的数字生命将面临"经济性死亡"

### 方案 D：用户数据一次性出售

- 描述：用户将数据打包一次性卖给数据经纪商
- 否决理由：违背主权原则。一次性出售后用户失去对数据的控制权，无法撤回授权。与 Project IV 追求的"持续可控的主权变现"根本矛盾

## 实施优先级

| 阶段 | 机制 | 优先级 |
|:---|:---|:---|
| **Spark MVP** | L0 成本意识路由（基础版：本地规则引擎替代简单 API 调用） | P0 |
| **Spark MVP** | 社区信誉积分（最简闭环：知识贡献→积分→可兑换额度） | P1 |
| **第二幕** | L0 成本意识路由（完整版：接入开源本地模型） | P0 |
| **第二幕** | 好友算力共享（信任域内的 Server 化身互助） | P1 |
| **第三幕** | 道器分成与教练模板（创作者经济基础） | P1 |
| **远期** | Vox 经纪人、第二大脑微特许 | P2 |
| **远期** | 主权广告网络、注意力池 | P3 |

## 后果

### 正面

- 用户使用 Project IV 不再是净支出，而是有可能实现收支平衡甚至盈利
- "节流、对冲、投资"三级模型覆盖了从轻度用户到重度用户的全谱系需求
- 所有机制不依赖代币、广告或数据一次性出售，与主权原则和合规红线一致
- L0 成本路由是纯技术解决方案，不依赖外部商业合作，可在 MVP 阶段立即实施

### 负面

- Vox 经纪人、第二大脑微特许等远期机制依赖外部商业合作方，市场培育需要时间
- 社区信誉积分在缺乏外部锚定的情况下，初期价值感知可能较弱
- Token 投资模式的有效性高度依赖 Vox 本身的智能水平——如果 Vox 不够强，用户无法获得高回报

### 缓解措施

- **早期聚焦 L0 成本路由**：这是最可控、最不依赖外部的节流手段。让用户首先感受到"AI 对话不再是烧钱"
- **社区积分轻量冷启动**：在 Spark MVP 中设计最简单的积分闭环（贡献知识块 → 获得积分 → 兑换社区内服务），不追求与法币锚定，只验证"贡献有回报"的行为模式
- **Vox 能力优先**：在推进 Token 投资模式之前，优先确保 Vox 在教练阶段的能力——只有真正帮用户提升决策质量的 AI，才能成为有价值的"生产工具"
- **远期机制的标准化准备**：在道器清单规范中预留"授权定价"字段，在外交协议中预留"数据授权"消息类型，为远期商业合作做好协议层面的准备
