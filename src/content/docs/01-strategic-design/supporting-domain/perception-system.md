## 1. Positioning

The Perception System is Vox's "senses." It evolves the consensus body from a question-answering machine that only responds to explicit commands, into a digital companion that understands user state and proactively provides support at the right moment in the right way.

The Perception System's position in the architecture:

- **Data Collection**: Standardized, desensitized sensor data streams provided by the edge environment where the Avatar resides
- **Data Fusion**: Fragmented data fused into high-level contextual states by the L0 layer's perception module
- **Decision Triggering**: Vox determines whether and how to intervene based on contextual states and the Principle Vault

The Perception System itself makes no decisions. It only answers "what state is the user in now" — whether to intervene, what to say, and how to say it are Vox's responsibilities.

## 2. Perception Principles

All perception behaviors follow four supreme guidelines. These are not functional requirements, but ethical constraints — they directly implement the sovereignty and trustworthiness principles.

### 2.1 Perception ≠ Surveillance

The system only collects the minimum data necessary to form context. Raw behavioral content is not recorded (e.g., browsed webpage titles, typed text, videos being watched). Data is desensitized on the edge before entering the perception fusion pipeline. Raw sensor data is discarded after use, not persisted or transmitted.

### 2.2 Silence Is a Virtue

Vox remains silent by default. The Perception System's purpose is not to "seize every opportunity to interact," but to not miss that one moment when the user truly needs help. Most of the time, the Perception System runs quietly, invisible to the user.

### 2.3 Empower, Don't Shape

The Perception System's goal is to help users better understand their own state and make higher-quality decisions — not to guide user behavior toward a predetermined direction. This means:

- The system can suggest "You've been working on this task for 2 hours, would you like a break?" but cannot say "You should take a break"
- The system can present attention heatmaps to help users reflect, but cannot use them as judgment criteria
- Any perception-triggered suggestions from Vox are labeled as "suggestions" and can be dismissed with one click

### 2.4 Full Informed Consent and Control

Users have complete control over all perception dimensions:

- **Right to Know**: Data sources, collection methods, and purposes for each perception dimension are clearly stated in the system
- **Right to Toggle**: Any dimension can be individually disabled; once disabled, no data is collected for that dimension
- **Right to Verify**: Users can view current perception state at any time ("What state does the system think I'm in now?")
- **Right to Delete**: High-level contextual states generated by perception fusion can be deleted and will not enter long-term memory

## 3. Perception Dimensions

The system fuses data streams from four dimensions to build a dynamic "context awareness" model. Data collection and desensitization for each dimension are completed on the edge.

### 3.1 Time Perception

| Item | Description |
|:---|:---|
| **Data Sources** | System clock, user-configured calendar (optional), historical behavior patterns (analyzed locally, not transmitted) |
| **Desensitization** | Does not read specific calendar content (e.g., "meeting with someone"), only extracts time slot tags |
| **Output** | Time slot labels ("Early morning 6-8", "Late night 23-05"), rhythm deviation markers (2 hours later than usual), periodic patterns ("Deep reading time every Sunday afternoon") |
| **Usage** | Vox adjusts interaction style based on time slot (quiet at night, normal during the day); gentle reminders when rhythm deviations are detected |

### 3.2 Attention Perception

| Item | Description |
|:---|:---|
| **Data Sources** | Active application window category (no title reading), keyboard/mouse input frequency patterns, current Avatar foreground/background state |
| **Desensitization** | Does not record "browsing website X", only activity categories ("Information retrieval", "Creation", "Communication", "Entertainment", "System operation"); does not record typed content, only input frequency and rhythm |
| **Output** | Attention state labels ("Focused" — continuous input, "Fragmented" — frequent window switching, "Stuck" — long idle or repeated modifications to same paragraph, "Flow" — stable browsing or reading rhythm) |
| **Usage** | "Stuck" state triggers gentle Vox inquiry; focus state maintains silence |

### 3.3 Location & Environment Perception

| Item | Description |
|:---|:---|
| **Data Sources** | Network type (WiFi/cellular/wired), connected device type, timezone |
| **Desensitization** | No GPS precise location, no WiFi SSID recording (only "known network / unknown network" marker), no IP address recording |
| **Output** | Environment labels ("Home", "Office", "Mobile", "Public place"), network stability assessment |
| **Usage** | Automatically lower default voice interaction volume in public places; simplify Vox output to quickly scannable short sentences when mobile; prioritize local processing when network is unstable |

### 3.4 Task Perception

| Item | Description |
|:---|:---|
| **Data Sources** | Current Pinned projects in Second Brain, active document metadata (type, tags, last modified time), today's focus in daily journal |
| **Desensitization** | Does not read document content, only accesses structured metadata layer |
| **Output** | Current task context ("Working on document drafting phase of Project A"), task switching frequency, Pin completion rate trends |
| **Usage** | When user frequently switches between projects, Vox can suggest re-evaluating today's Pin priorities; when Pin completion rate remains low, can gently ask next morning if plan adjustment is needed |

## 4. Perception Fusion

Desensitized data streams from each dimension are merged by the L0 layer's perception fusion module into **high-level contextual states**. Fusion is not simple label concatenation, but an inference process.

### 4.1 Fusion Logic

The fusion module takes outputs from the four dimensions as input and outputs one or more **scene hypotheses** with confidence levels. When multiple dimensions point to the same scene, confidence increases.

### 4.2 Fusion Examples

| Time | Attention | Environment | Task | Fusion Result | Vox Behavior |
|:---|:---|:---|:---|:---|:---|
| Late night | Focused | Home | Writing | **Discretionary creative time** (High confidence) | Complete silence, no disturbance |
| Late night | Fragmented | Home | No explicit task | **Possible bedtime relaxation** (Medium confidence) | Silent unless user initiates |
| Afternoon | Stuck | Office | Repeatedly editing same paragraph | **Possible perfectionism loop** (Medium confidence) | Optional inquiry: "Would you like another perspective on this text?" |
| Early morning | Focused | Home | 2/3 Pins completed | **Efficient morning work** (High confidence) | Silent, positive feedback upon completion |
| Evening | Fragmented | Mobile | Frequent project switching | **Possible decision fatigue or distraction** (Low confidence) | No intervention, pattern recorded for user's evening review |

### 4.3 Misjudgment Protection

Every scene label from the fusion module includes confidence. Only scenes with confidence ≥ medium are considered for Vox intervention. Low-confidence scenes are only recorded as "pattern data points" for user-initiated review reference, not triggering any proactive interaction.

Users can view "what state does the system think I'm in now" at any time and correct misjudgments. Each correction feeds back to the fusion module to adjust that user's state inference model.

## 5. Integration with Vox and Harness

### 5.1 Perception → Vox

Contextual states output by the perception module serve as additional input parameters during Vox reasoning — along with user input, conversation history, and Principle Vault — participating in the Multi-Perspective Parliament's reasoning process. When generating suggestions, the Emotive Brain in the parliament pays special attention to emotion and fatigue signals in the context.

### 5.2 Perception → Harness

Perception outputs also trigger Harness hard constraints:

- **Silence Constraint Linkage**: When perception determines "discretionary creative time," Harness automatically raises Vox's proactive intervention threshold — even if L2 thinks "user might need help," silence must be maintained
- **Time Constraint Linkage**: During late night hours, if perception system confirms user is indeed in "bedtime state" rather than "late-night work," Harness hard constraint prohibits Vox from initiating non-urgent interactions
- **Audit Protection**: Contextual states generated by perception system are also covered by Harness audit logs, allowing users to verify when and based on what context the system made what judgments

| Location & Environment Perception | On / Off | On |
| Task Perception | On / Off | On |

All control modifications take effect immediately without restart.
