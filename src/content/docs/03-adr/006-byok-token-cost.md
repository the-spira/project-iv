title: ADR 006 · BYOK Token Cost Sustainability Model
description: How the system mechanism ensures that users' BYOK token consumption costs less than the benefits or returns generated by that consumption.
---

## Decision Status

Accepted. Determined during Project IV economic sustainability discussions in 2026.

## Background

Project IV adopts a BYOK (Bring Your Own Key) model, where users provide their own API tokens to access the L3 external knowledge layer. This raises a core problem: users' token consumption is a continuous, deterministic economic cost. If using a digital life always results in net expenditure, the promises of sovereignty and equality lose their economic meaning — only those who can afford the ongoing cost can possess a digital life.

In the sustainability discussion, the core question was converged to:

> How can each user's token consumption cost be less than the benefit or return generated by that invocation?

## Decision

**Project IV adopts a "Three-Level Economic Benefit Model" to ensure BYOK cost sustainability: Token Throttling, Token Hedging, and Token Investment. These three levels progress according to the user's depth of use — from "spending less" to "earning back what was spent" to "spending is investing."**

### Level 1: Token Throttling — Keep benefits unchanged while costs approach zero

**Mechanism**: L0 intelligent routing + local open-source models.

Project IV's four-layer AI architecture acts as a natural cost throttle. The L0 layer assesses intent complexity before each user query. Simple tasks (summarization, classification, retrieval, sentiment analysis) are routed directly to local open-source small models or rule engines, with zero token consumption. Only deep reasoning, complex creation, and other tasks that cannot be completed locally invoke the costly L3 external API.

**Expected effect**: Over 80% of daily AI interactions incur no external token costs.

### Level 2: Token Hedging — Token consumption generates direct concurrent benefits

**Mechanism A · Vox Broker**:

When users are "doing things for themselves" (writing review journals, organizing knowledge), Vox simultaneously transforms these actions into external value:

- Review content is anonymized, structured, and generated into "experience knowledge blocks" in that domain
- Vox automatically matches knowledge blocks with external demand parties (AI training platforms, industry research, community knowledge pools) through the diplomacy protocol
- Upon successful matching, the user's token consumption for this session receives a return

**Mechanism B · Second Brain Micro-Franchise**:

Users mark high-quality knowledge blocks in their Second Brain (domain summaries, project reviews, principle templates) as "licensable," generating anonymized mirrors. External AI platforms pay per use — each time someone uses this knowledge block to train a model or as reference data, the user receives micro passive income. Token consumption is "production cost," licensing fees are "ongoing returns."

**Mechanism C · Community Internal Reputation Points**:

During the MVP stage, reputation points serve as the internal settlement medium. Users contribute anonymized knowledge blocks to earn points, which can be exchanged for API quota donations or curation services provided by other community members. This is a lightweight cold-start solution before external commercial partnerships mature.

### Level 3: Token Investment — Token as the cost of high-return production tools

**Scenario examples**:

- Independent developers using Vox to assist with outsourced projects — token cost as project expense, net income far exceeding token fees
- Creators using Vox to generate high-quality content listed on the Dao-Qi Marketplace, earning ongoing sales revenue
- Knowledge workers using Vox for deep analysis and decision support, producing higher-value analytical reports

At this level, users do not need complex hedging mechanisms — tokens themselves are the fuel for production tools, and the return rate depends on the user's own skills and creativity. Project IV's role is to ensure Vox is powerful enough to truly help users improve decision quality and output efficiency.

| Mechanism | Dependent Existing Architecture | Current Status |
|:---|:---|:---|
| L0 intelligent routing + cost awareness | L0-L3 layered architecture (designed) | "Cost" dimension needs to be added to L0 routing logic |
| Vox Broker | Diplomacy Protocol + Harness Layer + Perception System (designed) | Long-term, requires commercial partners |
| Second Brain Micro-Franchise | PARA structure + knowledge block versioning (designed) + Diplomacy Protocol | Long-term, requires standardized licensing agreements |
| Community Reputation Points | Dao-Qi ecosystem + Curator system (designed) | Medium-term, minimal closed loop can be designed in Spark |
| Token Investment | Vox Consensus Body (designed) | Depends on continuous Vox capability enhancement |

## Considered Alternatives

### Option A: Pure Advertising Model

- Description: Display ads to users, using ad revenue to cover token costs
- Rejected: The traditional advertising model relies on centralized collection and profiling of user data, conflicting with Project IV's sovereignty principles. A "sovereign advertising network" (client-side ad matching, data never leaves the device) could be explored long-term, but it is not a priority path for the MVP stage

### Option B: Token/Cryptocurrency Incentives

- Description: Issue an ecosystem token, incentivizing user behavior through "X-to-Earn" models
- Rejected: Clearly violates compliance red lines (the third red line established in ADR 004 — "no involvement in virtual currency transactions"). Financialization conflicts with Project IV's "Everyman" positioning and introduces regulatory risk

### Option C: Pure Donation/Sponsorship Model

- Description: Subsidize users' token costs through community donations or foundation sponsorships
- Rejected: Not scalable. Relying on a few people's goodwill to continuously subsidize the majority violates the "resilience" principle — donation sources are uncontrollable, and once cut off, users' digital lives face "economic death"

### Option D: One-Time User Data Sale

- Description: Users package and sell their data to data brokers in one transaction
- Rejected: Violates sovereignty principles. After a one-time sale, users lose control over their data and cannot revoke authorization. Fundamentally contradicts Project IV's pursuit of "continuously controllable sovereign monetization"

| Stage | Mechanism | Priority |

| **Spark MVP** | Community reputation points (minimal closed loop: knowledge contribution → points → redeemable quota) | P1 |
| **Act II** | L0 cost-aware routing (full: integrate local open-source models) | P0 |
| **Act II** | Friend compute sharing (trust domain Server avatar mutual aid) | P1 |
| **Act III** | Dao-Qi revenue sharing & coach templates (creator economy foundation) | P1 |
| **Long-term** | Vox Broker, Second Brain Micro-Franchise | P2 |
| **Long-term** | Sovereign advertising network, attention pool | P3 |

## Consequences

### Positive

- Using Project IV is no longer a net expense for users, but has the potential to break even or even generate profit
- The "Throttling, Hedging, Investment" three-level model covers the full spectrum of users from light to heavy
- All mechanisms do not rely on tokens, advertising, or one-time data sales, consistent with sovereignty principles and compliance red lines
- L0 cost routing is a pure technical solution that does not depend on external commercial partnerships and can be implemented immediately in the MVP stage

### Negative

- Long-term mechanisms such as Vox Broker and Second Brain Micro-Franchise depend on external commercial partners; market cultivation takes time
- Community reputation points may have weak perceived value in the early stages without external anchoring
- The effectiveness of the Token Investment model is highly dependent on Vox's intelligence level — if Vox is not strong enough, users cannot achieve high returns

### Mitigations

- **Early focus on L0 cost routing**: This is the most controllable, least externally dependent throttling method. Let users first feel that "AI conversations are no longer burning money"
- **Lightweight community points cold start**: Design the simplest points closed loop in Spark MVP (contribute knowledge blocks → earn points → redeem community services), without pursuing fiat anchoring, only validating the "contribution has rewards" behavior pattern
- **Vox capability priority**: Before advancing the Token Investment model, prioritize ensuring Vox's capabilities at the coach stage — only an AI that truly helps users improve decision quality can become a valuable "production tool"
- **Standardization preparation for long-term mechanisms**: Reserve "licensing pricing" fields in the Dao-Qi listing specification, reserve "data authorization" message types in the diplomacy protocol, preparing the protocol layer for future commercial partnerships
